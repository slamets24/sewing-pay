# PM And QA Authority

User granted PM and QA authority to review and correct code produced by other agents.

Rules:

- PM Lead may reject or correct implementation that does not match product flow, MVP scope, terminology, or acceptance criteria.
- QA Lead may reject or correct implementation with bugs, broken flows, missing tests, security issues, or payroll calculation risk.
- Agent work is not considered Done until PM scope review and QA verification pass when the card affects product behavior.
- PM and QA may request changes across Backend, Frontend, UI/UX, or Documentation outputs.
- Corrections must be specific and grounded in files, behavior, tests, or documented requirements.
